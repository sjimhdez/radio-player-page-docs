# Radio Player Page — Documentation

This repository contains the documentation for the **Radio Player Page** WordPress plugin. It is intended to be published as a documentation site (e.g. on GitHub Pages using Jekyll, MkDocs, or another static site generator). The documentation is available in three languages.

## Documentation by language

| Language | Description | Link |
|----------|-------------|------|
| **Documentation (English – US)** | Full documentation in American English. | [docs/en/]({{ site.baseurl }}/docs/en/) |
| **Documentación (Español – España)** | Documentación completa en castellano de España. | [docs/es/]({{ site.baseurl }}/docs/es/) |
| **Documentação (Português – Brasil)** | Documentação completa em português do Brasil. | [docs/pt/]({{ site.baseurl }}/docs/pt/) |

Each language folder contains an index and six topic pages: Installation, Configuration, Program Schedule, Visual Customization, Multiple Stations, and FAQ.

## Publishing on GitHub Pages

To publish this documentation on GitHub Pages:

1. Push this folder (or its contents) to a repository.
2. In the repository settings, enable **GitHub Pages** and choose the branch and folder (e.g. `/docs` or the root).
3. If you use Jekyll with `docs/` as the source directory, you may need to set `baseurl` in `_config.yml` to match your site URL (e.g. `/radio-player-page-docs`) so that asset paths resolve correctly.
4. Add the four screenshots described in [assets/images/README.md]({{ site.baseurl }}/assets/images/README.md) when available; the documentation references them so links will work once the image files are in place.
